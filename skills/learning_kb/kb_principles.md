# 全量知识库生成原则

## 设计思路：矩阵式场景覆盖

18 个主核心场景 + 100 个子情境，纵横交错。

### 词汇分配策略

- **通用词 (Top 300)**：在所有子情境中自然循环
- **场景词 (Next 700)**：分布在 100 个子情境中，平均每个子情境引入 ~7 个特定新词

### 核心逻辑要求

- **词汇分层**：1000 词按频率分为 10 组（V1-V10），每组 100 词
- **阶梯式递进**：
  - 第 1-10 子情境仅使用 V1
  - 第 11-20 子情境使用 V1 + V2
  - 第 21-30 子情境使用 V1-V3
  - 依此类推，新词占比 ≤ 15%
- **场景密度**：每个子情境 5-8 句对话
- **Krashen i+1**：每个对话中首次出现单词比例 ≤ 15%

### 本地化策略

- 标准通用英语作为底层字典
- Agent 在生成时根据用户目标地区替换 10% 为标准俚语

### 发音标注要求

每个核心例句包含：
1. 通用罗马音
2. 国际音标 (IPA)
3. LLM 描述性发音指导
